Variables y Constantes

¿Qué son las variables y las constantes?

Imagina que estás construyendo una casa. Los ladrillos son los elementos básicos que utilizas para construir las paredes. En programación, las variables y las constantes son como esos ladrillos: son los elementos fundamentales que utilizas para construir tus programas.

Variables: Son como cajas con un nombre específico donde puedes guardar diferentes tipos de información, como números, letras o palabras. El contenido de estas cajas puede cambiar a lo largo de la ejecución del programa. Constantes: Son como cajas que tienen un valor fijo y no pueden cambiar una vez que han sido asignadas. Son como etiquetas que indican un valor que no debe modificarse.

¿Para qué sirven?

Variables:

Almacenar datos que pueden cambiar durante la ejecución del programa.
Realizar cálculos y operaciones.
Controlar el flujo de un programa.

Constantes:

Representar valores que no cambian, como números pi o la velocidad de la luz.
Hacer el código más legible y mantenible.
Evitar errores al modificar accidentalmente un valor importante.

Tipos de Datos

¿Qué es un tipo de dato?

Imagina que tienes cajas de diferentes tamaños y formas. En una caja pequeña puedes guardar un anillo, en una caja mediana un libro y en una caja grande una pelota de baloncesto. Cada caja representa un tipo de dato diferente, y el objeto que guardas dentro de ella debe tener el tamaño y la forma adecuados. En C++, los tipos de datos definen el tamaño y el rango de valores que una variable puede almacenar. Esto le indica al compilador cómo interpretar los datos y qué operaciones se pueden realizar con ellos.

Tipos de datos básicos

INT

Representa números enteros, tanto positivos como negativos (ej: 5, -10, 0).

FLOAT

Representa números de punto flotante (con decimales), con una precisión limitada (ej: 3.14, -2.5).

DOUBLE

Representa números de punto flotante con mayor precisión que float (ej: 3.14159265).

CHAR

Representa un solo carácter, como una letra, un número o un símbolo especial (ej: 'a', 'Z', '9').

BOOL

Representa valores booleanos, que pueden ser true (verdadero) o false (falso).

Condicionales

¿Qué son los condicionales?

Imagina que estás conduciendo un coche y llegas a un cruce. Tienes que tomar una decisión: ¿girar a la izquierda, a la derecha o seguir recto? En programación, los condicionales nos permiten tomar decisiones similares dentro de nuestros programas.

Los condicionales nos permiten ejecutar diferentes bloques de código dependiendo de si se cumple o no una determinada condición. Son como bifurcaciones en un camino: si se cumple una condición, el programa sigue por un camino; si no se cumple, sigue por otro.

La estructura básica del condicional IF

if (condición) {
// Código a ejecutar si la condición es verdadera
}

Condición: Es una expresión que se evalúa como verdadera o falsa.
Bloque de código: Es el conjunto de instrucciones que se ejecutan si la condición es verdadera. Bloque de código

El condicional else

if (condición) {
// Código a ejecutar si la condición es verdadera
} else {
// Código a ejecutar si la condición es falsa
}

Ciclos o Bucles

¿Qué son los ciclos en programación?

Imagina que quieres imprimir los números del 1 al 100 en la pantalla. ¿Te imaginas escribiendo cada número en una línea diferente? Sería muy tedioso y propenso a errores. Los ciclos nos permiten automatizar tareas repetitivas como esta.

Un ciclo, también conocido como bucle, es una estructura de control que permite ejecutar un bloque de código repetidamente mientras se cumpla una determinada condición.

Tipos de ciclos

for: Se utiliza cuando se conoce de antemano el número de iteraciones.
while: Se utiliza cuando la condición de finalización del ciclo se evalúa al principio de cada iteración.
do-while: Es similar al ciclo while, pero la condición se evalúa al final de cada iteración, garantizando que el cuerpo del ciclo se ejecute al menos una vez.

Funciones

¿Qué es una funcion?

Imagina que tienes una receta de cocina. La receta te indica los ingredientes y los pasos que debes seguir para preparar un plato. En programación, una función es como una receta: es un bloque de código que realiza una tarea específica. Las funciones nos permiten dividir nuestro programa en partes más pequeñas y manejables, haciendo que el código sea más fácil de leer, entender y mantener.

Estructura de una función

Tipo_de_dato: El tipo de dato que la función devolverá.
Nombre_de_la_función: Un nombre que identifica a la función.
Parámetros: Los datos que se le pasan a la función para que realice su tarea.
Cuerpo de la función: El conjunto de instrucciones que la función ejecuta.
Return: La palabra clave que se utiliza para devolver un valor desde la función.

Programacion Orientada a Objetos

¿Qué es la POO?

La Programación Orientada a Objetos (POO) es un paradigma de programación que se basa en la idea de organizar el código en torno a objetos. Un objeto es una entidad que tiene propiedades (atributos) y comportamientos (métodos). Por ejemplo, un coche es un objeto que tiene propiedades como color, marca y modelo, y comportamientos como acelerar, frenar y girar.

Conceptos fundamentales de la POO

Clase: Es el plano o molde a partir del cual se crean los objetos. Define los atributos y métodos que compartirán todos los objetos de esa clase.
Objeto: Es una instancia de una clase. Cada objeto tiene sus propios valores para los atributos.
Atributos: Son las características o propiedades de un objeto. Por ejemplo, el color y el tamaño de un círculo.
Métodos: Son las acciones que un objeto puede realizar. Por ejemplo, el método calcularArea() para un círculo.
Encapsulamiento: Consiste en ocultar los detalles internos de un objeto y exponer solo una interfaz para interactuar con él. Esto protege los datos y evita modificaciones no deseadas.
Herencia: Permite crear nuevas clases (clases derivadas) a partir de clases existentes (clases base), heredando sus atributos y métodos. Esto promueve la reutilización de código.
Polimorfismo: Permite que objetos de diferentes clases puedan ser tratados como si fueran de una misma clase, siempre y cuando tengan métodos con el mismo nombre.
